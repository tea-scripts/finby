# Finby — Deployment Runbook

Production topology (one-time setup, then push-to-deploy):

```
finby.app        → marketing landing      (Vercel — separate repo, unchanged)
chat.finby.app   → apps/web (chat app)     (Vercel)
api.finby.app    → apps/api (NestJS)        (Render, Starter $7/mo) ── Key Value/Redis (Render, free)
                                            └── Postgres (Prisma Postgres, existing plan)
```

Auth tokens travel in the `Authorization` header (not cookies), so cross-subdomain
works with just CORS — no cookie-domain config. The API's CORS origin is `WEB_URL`.

---

## 1. Database — Prisma Postgres
1. Prisma console → **Finby** project → **Databases → +** → create `finby-app`, in the
   **same region** you'll use for Render.
2. **Connection strings → Direct** → copy the `postgres://…` string and append
   **`?connection_limit=5`**. This is `DATABASE_URL`.
   - Use the **direct** string, not the `prisma+postgres://` Accelerate one (the app is plain Prisma 5).
   - Migrations run automatically via the API's `preDeployCommand` (`prisma migrate deploy`).
   - No seeding in prod — default categories are created at registration.

## 2. API + Redis — Render (via Blueprint)
1. Render Dashboard → **New → Blueprint** → pick this repo → it reads `render.yaml`
   (creates `finby-api` web service + `finby-redis` Key Value).
2. Edit `render.yaml`'s `region:` (both services) to match the Prisma DB region first if not `oregon`.
3. Fill the `sync: false` secrets in the dashboard (see the env table below) — most importantly `DATABASE_URL`.
4. After first deploy: **Settings → Custom Domains** confirms `api.finby.app`; add the
   shown CNAME at your DNS (step 4).
5. Health check: `GET https://api.finby.app/api/v1/health` → `200`.

## 3. Web — Vercel
1. Vercel → **Add New → Project** → import this repo.
2. **Root Directory:** `apps/web`.
3. **Build Command (override):** `cd ../.. && pnpm turbo run build --filter=finby-web`
   (builds `@finby/shared` first, then the web app).
4. **Environment Variable:** `NEXT_PUBLIC_API_URL=https://api.finby.app/api/v1`.
5. **Domains:** add `chat.finby.app` → add the CNAME Vercel shows (step 4).

## 4. DNS (Namecheap, finby.app) — apex unchanged
| Host   | Type  | Target                              |
|--------|-------|-------------------------------------|
| `chat` | CNAME | (value Vercel shows for the domain) |
| `api`  | CNAME | (value Render shows for the domain) |

## 5. Stripe webhook (when billing goes live)
Stripe → Webhooks → endpoint `https://api.finby.app/api/v1/webhooks/stripe` →
copy the signing secret → set `STRIPE_WEBHOOK_SECRET` on Render.
(See "Payments" note below — a Merchant of Record may replace Stripe.)

## Post-deploy smoke test
1. `GET /api/v1/health` → 200.
2. On `chat.finby.app`: register → land in chat → send "spent 12 on lunch" (exercises Anthropic + DB + Redis).
3. Dashboard + Transactions load.
4. (When billing live) trigger a test webhook → tier flips.

## Notes / gotchas
- **Co-locate** Render region ↔ Prisma DB region (query-heavy server; cross-region latency hurts).
- `@finby/shared` is consumed from its built `dist/`, so builds must run via `turbo`
  (handled in both the Render and Vercel build commands).
- Native deps (`bcrypt`, `prisma`, `esbuild`, `sharp`) are allow-listed in root
  `pnpm.onlyBuiltDependencies`, so `pnpm install` compiles them on Render.
- Render Key Value is private (`ipAllowList: []`) — only the API reaches it.

## Payments (PH/Nigeria, global acceptance)
Stripe direct isn't available for KYC in PH/Nigeria. Use a **Merchant of Record**
(Lemon Squeezy or Paddle): they're the legal seller, accept cards globally, handle
worldwide sales tax/VAT, and pay you out (PayPal/bank — PH path is cleanest). Add a
`LemonSqueezyProvider` to the existing `BillingProvider` port when ready. Alternative
for full Stripe later: Stripe Atlas (US LLC). Deployment is **not blocked** — the app
FREE-synthesizes subscriptions and tier-gates until billing is wired.
