# Finby ‚Äî Deployment Runbook

Production topology (one-time setup, then push-to-deploy):

```
finby.app        ‚Üí marketing landing      (Vercel ‚Äî separate repo, unchanged)
chat.finby.app   ‚Üí apps/web (chat app)     (Vercel)
api.finby.app    ‚Üí apps/api (NestJS)        (Render, Starter $7/mo) ‚îÄ‚îÄ Key Value/Redis (Render, free)
                                            ‚îî‚îÄ‚îÄ Postgres (Prisma Postgres, existing plan)
```

Auth tokens travel in the `Authorization` header (not cookies), so cross-subdomain
works with just CORS ‚Äî no cookie-domain config. The API's CORS origin is `WEB_URL`.

---

## 1. Database ‚Äî Prisma Postgres
1. Prisma console ‚Üí **Finby** project ‚Üí **Databases ‚Üí +** ‚Üí create `finby-app`, in the
   **same region** you'll use for Render.
2. **Connection strings ‚Üí Direct** ‚Üí copy the `postgres://‚Ä¶` string and append
   **`?connection_limit=5`**. This is `DATABASE_URL`.
   - Use the **direct** string, not the `prisma+postgres://` Accelerate one (the app is plain Prisma 5).
   - Migrations run automatically via the API's `preDeployCommand` (`prisma migrate deploy`).
   - No seeding in prod ‚Äî default categories are created at registration.

## 2. API + Redis ‚Äî Render (via Blueprint)
1. Render Dashboard ‚Üí **New ‚Üí Blueprint** ‚Üí pick this repo ‚Üí it reads `render.yaml`
   (creates `finby-api` web service + `finby-redis` Key Value).
2. Edit `render.yaml`'s `region:` (both services) to match the Prisma DB region first if not `oregon`.
3. Fill the `sync: false` secrets in the dashboard (see the env table below) ‚Äî most importantly `DATABASE_URL`.
4. After first deploy: **Settings ‚Üí Custom Domains** confirms `api.finby.app`; add the
   shown CNAME at your DNS (step 4).
5. Health check: `GET https://api.finby.app/api/v1/health` ‚Üí `200`.

## 3. Web ‚Äî Vercel
1. Vercel ‚Üí **Add New ‚Üí Project** ‚Üí import this repo.
2. **Root Directory:** `apps/web`.
3. **Build Command (override):** `cd ../.. && pnpm turbo run build --filter=finby-web`
   (builds `@finby/shared` first, then the web app).
4. **Environment Variable:** `NEXT_PUBLIC_API_URL=https://api.finby.app/api/v1`.
5. **Domains:** add `chat.finby.app` ‚Üí add the CNAME Vercel shows (step 4).

## 4. DNS (Namecheap, finby.app) ‚Äî apex unchanged
| Host   | Type  | Target                              |
|--------|-------|-------------------------------------|
| `chat` | CNAME | (value Vercel shows for the domain) |
| `api`  | CNAME | (value Render shows for the domain) |

## 5. Stripe webhook (when billing goes live)
Stripe ‚Üí Webhooks ‚Üí endpoint `https://api.finby.app/api/v1/webhooks/stripe` ‚Üí
copy the signing secret ‚Üí set `STRIPE_WEBHOOK_SECRET` on Render.
(See "Payments" note below ‚Äî a Merchant of Record may replace Stripe.)

## Post-deploy smoke test

2. On `chat.finby.app`: register ‚Üí land in chat ‚Üí send "spent 12 on lunch" (exercises Anthropic + DB + Redis).
3. Dashboard + Transactions load.
4. (When billing live) trigger a test webhook ‚Üí tier flips.

## Notes / gotchas
- **Co-locate** Render region ‚Üî Prisma DB region (query-heavy server; cross-region latency hurts).
- `@finby/shared` is consumed from its built `dist/`, so builds must run via `turbo`
  (handled in both the Render and Vercel build commands).
- Native deps (`bcrypt`, `prisma`, `esbuild`, `sharp`) are allow-listed in root
  `pnpm.onlyBuiltDependencies`, so `pnpm install` compiles them on Render.
- Render Key Value is private (`ipAllowList: []`) ‚Äî only the API reaches it.

## Payments (PH/Nigeria, global acceptance) ‚Äî Lemon Squeezy (Merchant of Record)
Stripe direct isn't available for KYC in PH/Nigeria, so the app uses a **Merchant of
Record**: Lemon Squeezy is the legal seller, accepts cards globally, remits worldwide
sales tax/VAT, and pays you out (PayPal/bank ‚Äî PH path is cleanest). The
**`LemonSqueezyProvider` is implemented** on the `BillingProvider` port.

To go live:
1. Lemon Squeezy ‚Üí create the store + a **subscription product variant per tier**
   (Pro / Premium / Family).
2. Set on Render: `LEMONSQUEEZY_API_KEY`, `LEMONSQUEEZY_STORE_ID`,
   `LEMONSQUEEZY_WEBHOOK_SECRET`, and `LEMONSQUEEZY_VARIANT_PRO/PREMIUM/FAMILY`.
3. LS ‚Üí Webhooks ‚Üí endpoint **`https://api.finby.app/api/v1/webhooks/lemonsqueezy`**
   (subscribe to the `subscription_*` events) ‚Üí put the signing secret in
   `LEMONSQUEEZY_WEBHOOK_SECRET`.
4. The web checkout calls `POST /workspaces/:id/subscription/checkout` with
   `{ tier, provider: "LEMONSQUEEZY" }` (now the default).

Deployment is **not blocked** by this ‚Äî until the keys are set, the app
FREE-synthesizes subscriptions and tier-gates. (Full Stripe later: Stripe Atlas / US LLC.)
